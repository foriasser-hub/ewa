Le terme "**vibe coding**" est partout depuis quelques mois. Il désigne une nouvelle façon de créer des logiciels : vous décrivez ce que vous voulez, et une intelligence artificielle écrit le code pour vous, à vos côtés. Mais ce n'est pas magique non plus. Démêlons le sujet.

## Vibe coding ≠ "no code"

Beaucoup confondent les deux. Voici la différence :

- **No code** : vous assemblez des blocs visuels prédéfinis (Webflow, Bubble, Notion). Vite, mais limité dans la personnalisation.
- **Vibe coding** : vous écrivez en langage naturel ce que vous voulez, l'IA produit du **vrai code** (HTML, JavaScript, Python…). Aussi simple à démarrer, mais sans limite de personnalisation.

L'avantage du vibe coding, c'est que **votre projet n'est pas enfermé dans un outil**. Le code généré est standard et peut évoluer.

## Comment ça marche concrètement

Imaginez que vous voulez un site vitrine pour votre activité freelance. Voici votre dialogue avec l'assistant :

> **Vous** : Crée-moi un site vitrine en français, avec une page d'accueil, une page services et un formulaire de contact. Style moderne, palette bleu marine et blanc.
>
> **L'assistant** : (génère la structure et les pages, vous les montre)
>
> **Vous** : Sur la page services, j'aimerais 3 cards au lieu de 4, et change le titre par "Mes prestations".
>
> **L'assistant** : (modifie en direct)
>
> **Vous** : Parfait. Tu peux le déployer en ligne ?
>
> **L'assistant** : (configure Vercel, vous donne l'URL)

En une heure, vous avez un site en ligne. Sans avoir écrit une ligne de code seul·e.

## Ce que vous apprenez quand vous vibe-codez

C'est la partie souvent oubliée : en travaillant avec l'IA, **vous apprenez** progressivement :

- À **structurer une idée** en exigences claires (un mini cahier des charges).
- À **lire** du code généré, même si vous ne l'auriez pas écrit vous-même.
- À **dialoguer** avec une IA technique (un skill de plus en plus recherché en entreprise).
- À **déboguer** : quand quelque chose ne marche pas, vous apprenez à expliquer le problème.

C'est une compétence en soi, qu'on pourrait appeler la "**maîtrise du brief technique**".

## Les limites à connaître

Le vibe coding n'est pas magique :

- Pour un projet simple à moyen, ça marche très bien.
- Pour un produit complexe (sécurité critique, perfs élevées, équipe de 50 devs), il faut quand même des développeurs humains qui supervisent.
- Le code généré peut contenir des bugs subtils — d'où l'intérêt de tester systématiquement.
